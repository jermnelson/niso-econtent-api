Personal patron information

Comments from Best Practice Review Topic ID #21

No personal patron information should move from the identity provider (or ILS or discovery layer) to a third-party eContent vendor. That way there's no potential for abuse.

FASTEN Alignment with NCIP

  • A POST request with a NCIP User Id or Authentication Input with optional data is restricted to outputs defined in 5.3.4 Lookup User Service.

FASTEN Alignment with LCF

GET request to return E03 Patron entity (in XML)

FASTEN Alignment with BIC Library Web Services

FASTEN Alignment with OpenID

  • A GET request to the system with either a code, token, or id_token to an authorization server will return a access_token, token_type, id_token, and an optional expires_in. From this 5. Definitions of Multiple-Valued Response Type Combinations. No other personal information is transmitted to the vendor.

Not applicable
